# Implementation Plan: Physical AI & Humanoid Robotics Textbook

**Branch**: `001-book-master-plan` | **Date**: 2026-02-02 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-book-master-plan/spec.md`

## Summary

Build a comprehensive online textbook for a 13-week Physical AI & Humanoid Robotics course using Docusaurus v3. The site features a dashboard-style homepage with module cards, nested collapsible sidebar navigation, full educational metadata in frontmatter, embedded code examples with syntax highlighting, and automated deployment to GitHub Pages.

## Technical Context

**Language/Version**: TypeScript 5.x (Docusaurus config), MDX (content)
**Primary Dependencies**: Docusaurus 3.x, React 18.x, @docusaurus/preset-classic, @easyops-cn/docusaurus-search-local, @docusaurus/theme-mermaid
**Storage**: Static files (Markdown/MDX), no database
**Testing**: yarn build (compilation), Lighthouse CI (performance), linkinator (broken links)
**Target Platform**: GitHub Pages (static hosting), browsers (Chrome, Firefox, Safari, Edge)
**Project Type**: Static documentation site (Docusaurus)
**Performance Goals**: Lighthouse Performance ≥90, page load <3s, SEO score 100
**Constraints**: GitHub Pages hosting limits, static-only (no server-side rendering)
**Scale/Scope**: 30+ pages, 4 modules, 100+ glossary terms, ~25,000 words total

## Complexity Tracking

> No constitution violations requiring justification.

| Decision | Rationale | Alternative Considered |
|----------|-----------|------------------------|
| Embedded code snippets | Simpler maintenance, better search indexing | External repo with imports (more complex sync) |
| Local search plugin | No external service dependency, works offline | Algolia DocSearch (requires application/approval) |
| Autogenerated sidebar | Filesystem-based, less manual maintenance | Fully manual sidebars.ts (more control but tedious) |
