---

## Sidebar Configuration Approach

Use **autogenerated sidebar** with `_category_.json` files for category metadata. The sidebar is automatically generated from the filesystem structure in `docs/`.

### sidebars.ts Configuration

```typescript
import type {SidebarsConfig} from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  tutorialSidebar: [
    'intro',
    {
      type: 'autogenerated',
      dirName: '.',
    },
  ],
};

export default sidebars;
```

## Navigation Behavior

### Default State
- Introduction section: **Expanded** (first-time visitors see content immediately)
- All modules: **Collapsed** (reduces visual clutter, users expand as needed)
- Hardware Guide: **Collapsed**
- Appendices: **Collapsed**

### Mobile Behavior
- Sidebar hidden by default
- Hamburger menu toggles sidebar
- Touch-friendly tap targets (44px minimum)
- Swipe gestures supported

### Keyboard Navigation
- Tab through sidebar items
- Enter/Space to expand/collapse categories
- Arrow keys for navigation within category

---

## Implementation Notes

1. **File naming**: All markdown files use kebab-case (e.g., `core-concepts.md`)
2. **Index files**: Each category has an `index.md` that serves as the category landing page
3. **Sidebar position**: Defined in frontmatter of each page, determines order within category
4. **Custom props**: Available for React components to render week badges, prerequisite indicators
